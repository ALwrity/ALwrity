# SSOT: Content Calendar Vision

## The Narrative

> *"Your strategy tells us what matters. We built a calendar around it. Here's what you need to do today, with AI reasoning for each task."*

This is the core demo promise of ALwrity's Marketing OS. The Content Calendar is the central hinge — it translates strategic intent into daily execution. It is the bridge between **what you want to achieve** (strategy) and **what you need to do today** (workflow).

## 2. Strategic Principles

### 2.1 Calendar is an AI Artifact, Not a Manual Spreadsheet
The Content Calendar is generated by AI from the Content Strategy. The user does not fill a spreadsheet — they review and approve an AI-generated plan. Every event carries its strategic context: which pillar it serves, what keywords it targets, what angle it takes, which audience segment it addresses.

### 2.2 The Workflow is the Calendar's Execution Arm
The Today Workflow does not duplicate the calendar. It reads the calendar's events for today and presents them as actionable tasks. The workflow is the "now" view of the calendar. Strategic richness (pillar name, keyword targets, content angle) flows through to the task card so the user understands *why* this task matters today.

### 2.3 AI Reasoning is Attached to Every Task
Each task shows not just "what to do" but "why now." Calendar-sourced tasks show: "From your Q1 Content Strategy — AI & Automation pillar. This topic fills the gap in your tutorial content." Agent-sourced tasks show: "ContentGapRadar identified this opportunity based on competitor analysis."

### 2.4 The Loop is Closed
Content performance data feeds back into strategy refinement and future calendar generations. The system learns what resonates and adjusts.

## 3. The 12-Step Pipeline (Intent)

The pipeline transforms strategy data into a structured, day-level content plan.

| Phase | Step | What It Produces | Status |
|---|---|---|---|
| **Foundation** | 1: Content Strategy Analysis | Structured analysis of the active strategy — pillars, goals, KPIs | ✅ Real data |
| | 2: Gap Analysis & Opportunity | Content gaps, keyword opportunities, competitor blind spots | ✅ Real data |
| | 3: Audience & Platform Strategy | Audience segments, platform priorities, content preferences | ✅ Real data |
| **Structure** | 4: Calendar Framework & Timeline | Monthly/weekly framework, content frequency, publishing cadence | ✅ Real AI |
| | 5: Content Pillar Distribution | Weight distribution of pillars across the timeline | ✅ Real AI |
| | 6: Platform-Specific Strategy | Per-platform content mix, optimal timing, format strategy | ✅ Real AI |
| **Content** | 7: Weekly Theme Development | Weekly themes, focus areas, primary keywords per week | ✅ Real AI |
| | 8: Daily Content Planning | Day-level schedule: what content, which platform, at what time | ⚠️ Runs, quality score hardcoded |
| | 9: Content Recommendations | Additional content suggestions, gap-filling topics | ❌ Blocked by Step 8 issues |
| **Optimization** | 10: Performance Optimization | Engagement predictions, reach estimates, optimization tips | ❌ Blocked |
| | 11: Strategy Alignment Validation | Quality gates, pillar coverage check, audience alignment | ❌ Blocked |
| | 12: Final Calendar Assembly | Complete calendar with metadata, quality scores, validation | ❌ Blocked |

### 3.1 What Each Step Contributes to the Final CalendarEvent

When Step 8 produces a daily content item, the event carries:

```
CalendarEvent {
  title: "How to Implement AI in Your Marketing Strategy"
  content_type: "blog_post"
  platform: "blog"
  strategy_id: 42
  scheduled_date: "2025-02-03"
  
  // Strategic context from upstream steps:
  content_pillar: "AI & Automation"
  target_keywords: ["AI marketing", "implementation guide"]
  content_angle: "Practical implementation steps with real examples"
  target_audience: "Marketing professionals"
  estimated_engagement: 0.85
  
  // Action routing:
  action_url: "/blog-writer"
}
```

The `metadata_json` field on `DailyWorkflowTask` should carry the strategic context for display in the task card.

## 4. What the User Sees (Demo Flow)

### Step 1: Complete Onboarding
- Enter website, API keys
- AI analyzes brand, audience, competitors
- Personas generated

### Step 2: Create Content Strategy
- AI auto-fills 30+ fields from onboarding data
- User reviews and refines strategic inputs across 5 categories
- AI generates comprehensive analysis (strategic scores, market positioning, risks, opportunities)
- User activates the strategy

### Step 3: Calendar Auto-Generates (THE KEY MOMENT)
- On strategy activation, the 12-step pipeline starts automatically
- User sees real-time progress across the 12 steps (modal with StepProgressTracker)
- Calendar events populate the Calendar tab
- **Demo line**: "The system just built your content calendar for the next month — aligned with your strategy, your audience, and your business goals."

### Step 4: Today's Workflow Reflects the Calendar
- The Generate pillar shows today's calendar events as tasks
- Each task card shows:
  - Title and description (from the event)
  - "From your Content Calendar" badge (blue)
  - Strategy context: e.g., "AI & Automation pillar · Q1 Strategy"
  - Estimated time and engagement prediction
- Other pillars show agent-suggested tasks (plan, analyze, etc.)

### Step 5: Click a Task → Writer Pre-Filled
- Calendar topic, description, and strategic context flow into the writer
- AI generates first draft aligned with brand voice
- User refines and saves

### Step 6: Save → Calendar Published → Task Complete
- Content saved to Asset Library
- Calendar event marked as published
- Workflow task marked as completed
- User navigates back to workflow automatically

## 5. Key Gaps to Close

### Gap 1: Auto-Trigger from Strategy Activation
**Current**: After strategy creation, frontend switches to Calendar Wizard tab but user must click "Generate Calendar."  
**Target**: Strategy activation (`status = 'active'`) triggers the 12-step pipeline automatically.  
**Files involved**: `ActiveStrategyService`, strategy activation endpoint, `CalendarGenerationService.start_orchestrator_generation`

### Gap 2: Strategic Context in Workflow Tasks
**Current**: `_generate_calendar_event_plan()` only reads basic fields from CalendarEvent. Workflow task cards show title + description only.  
**Target**: CalendarEvent's `metadata_json` (or equivalent) carries content_pillar, target_keywords, content_angle, estimated_engagement. Workflow task card displays this strategic context.  
**Files involved**: `today_workflow_service.py:_generate_calendar_event_plan()`, `EnhancedTodayModal.tsx` task card rendering

### Gap 3: Step 8 Quality Score + Step 9-12 Completion
**Current**: Step 8 runs but hardcodes `qualityScore: 0.9`. Steps 9-12 are blocked.  
**Target**: Step 8 surfaces the real computed quality score. Steps 9-12 are implemented and pass validation.  
**Files involved**: `step8_implementation.py`, `step9_content_recommendations/`, `step10_performance_optimization/`, `step11_strategy_alignment/`, `step12_final_assembly/`

### Gap 4: Agent Committee Generate Proposals
**Current**: Filtered out when calendar events exist.  
**Target**: Merged alongside calendar tasks in the generate pillar. Calendar events take priority ordering but agent suggestions appear as additional tasks with "AI-suggested" badge.  
**Files involved**: `today_workflow_service.py:get_or_create_daily_workflow_plan()`

### Gap 5: Facebook/SEO/Video Action URLs
**Current**: `_resolve_calendar_action_url()` returns None for non-blog/linkedin content types. Events silently dropped.  
**Target**: Calendar events of any type produce a workflow task (even if the action is "Create in [platform]" opening the relevant tool).  
**Files involved**: `today_workflow_service.py:_resolve_calendar_action_url()`

### Gap 6: Closed-Loop Analytics (Future)
**Current**: No performance data feeds back into strategy or calendar generation.  
**Target**: Content engagement/performance metrics inform the next calendar generation — underperforming topics are deprioritized, winning formats are emphasized.

## 6. Implementation Roadmap

### Phase 1 — Fix the Pipeline (the 12 steps)
1. Fix Step 8: Surface real quality score instead of hardcoded 0.9
2. Unblock Steps 9-12: Implement content recommendations, performance optimization, strategy alignment, final assembly
3. Verify complete pipeline produces valid CalendarEvent records with full metadata

### Phase 2 — Auto-Trigger from Strategy
1. Add auto-trigger in strategy activation flow (backend: after `status = 'active'`, call `start_orchestrator_generation`)
2. Add strategy activation → calendar generation → notification to frontend
3. Show progress modal with the 12-step tracker

### Phase 3 — Strategic Context in Workflow
1. Extend `CalendarEvent` model or ensure `metadata_json` carries strategy context
2. Update `_generate_calendar_event_plan()` to pass through the extra fields
3. Update `EnhancedTodayModal.tsx` task card to show pillar, keywords, angle, predicted engagement

### Phase 4 — Agent Generate Proposals
1. Change merge logic to include agent generate tasks (not filter them out)
2. Add "AI-suggested" badge to distinguish from calendar-sourced tasks
3. Add deduplication by semantic similarity (not just title)

### Phase 5 — All Content Types
1. Add action URLs for facebook, seo_page, video content types
2. Ensure every calendar event maps to a workflow task

## 7. Success Criteria

The system is complete when:

1. **Strategy activation** auto-generates a full content calendar with zero clicks
2. **Calendar events** carry strategic context (pillar, keywords, angle, audience, predicted engagement)
3. **Workflow tasks** display this strategic context — user sees *why* each task matters
4. **Agent generate suggestions** appear alongside calendar tasks
5. **All content types** produce actionable workflow tasks
6. **Completed tasks** mark calendar events as published and update the store
7. **Demo flows end-to-end** without manual intervention after strategy activation
