# Quality Audit Panel

The **QualityAuditPanel** is the front-end component that renders the ContentGuardianAgent's committee audit report. It appears below the Committee Summary on the Team Activity page.

## Data Source

The panel reads the most recent `quality_audit` event from the `useAgentHuddleFeed` events array:

```typescript
const report = events
  .find((e) => e.event_type === 'quality_audit')
  ?.payload as AuditReport;
```

If no `quality_audit` event exists, the panel is hidden entirely.

## Audit Report Structure

```typescript
interface AuditReport {
  health_score: number;          // 0–100
  verdict: string;               // Human-readable summary
  agent_critiques: AgentCritique[];
  coverage_gaps: CoverageGap[];
  overstuffed_pillars?: CoverageGap[];
  overlaps: Overlap[];
  alerts: AuditAlert[];
  audit_timestamp: string;
}
```

## Visual Sections

### Health Ring

An SVG circular gauge rendering the `health_score` on a 0–100 scale:

| Score Range | Colour | Label |
|---|---|---|
| 80–100 | Green (`#4caf50`) | Healthy |
| 50–79 | Amber (`#ff9800`) | Needs review |
| 0–49 | Red (`#f44336`) | Failing |

The ring uses `strokeDasharray` and `strokeDashoffset` with CSS transitions for smooth animation.

### Verdict Text

A one-line plain-language summary generated by `_verdict_text()` in the ContentGuardianAgent. Examples:

- *"Committee is performing well — all agents submitting quality proposals with good coverage."*
- *"2 agent(s) need attention: SEO Optimization, Competitor Response — Missing coverage: engage"*
- *"1 agent(s) showing issues: Strategy Architect — Minor issues detected."*

### Agent Critiques (collapsible)

Each committee agent receives an `AgentCritiqueCard` showing:

| Element | Source | Display |
|---|---|---|
| Agent label | `critique.label` | Bold text header |
| Acceptance chip | `critique.accepted / critique.total_proposals` | Green (rate >50%) or Red chip |
| Score chip | `critique.score / 100` | Colour-coded by health |
| Score bar | `LinearProgress` at `critique.score` | Colour-matched progress bar |
| Summary | `critique.summary` | One-line plain text |
| Issue list | `critique.issues[]` | Expandable, each with severity icon and detail items |

#### Issue Types

| Issue Type | Severity Icon | Detail Keys |
|---|---|---|
| `weak_reasoning` | Info / Warning / Error | `title`, `reasoning`, `score` |
| `poor_priority` | Warning | `title`, `pillar`, `priority`, `note` |
| `off_pillar` | Info | `title`, `proposed_pillar`, `expected_pillar`, `note` |
| `rejected_proposals` | Warning / Error | `title`, `reason` |

Each issue row shows the first 2 detail items, with a "+N more" overflow indicator.

### Coverage Gaps (collapsible)

When one or more pillars received zero proposals, a collapsible section lists each gap:

| Element | Display |
|---|---|
| Warning icon | Orange `WarningAmber` icon |
| Summary | Plain text (e.g. *"Pillar 'engage' has no proposals"*) |
| Pillar chip | `pillar: engage` |

### Overlaps (collapsible)

When two or more agents propose tasks with the same title, a collapsible section lists each overlap:

| Element | Display |
|---|---|
| Info icon | Blue `Info` icon |
| Summary | Plain text (e.g. *"2 agents proposed 'Refresh SEO basics': SEO Specialist, Competitor Analyst"*) |

### Alert Badge

If `report.alerts.length > 0`, a red chip in the header shows the alert count. These alerts are also persisted as `AgentAlert` rows (see [Alert System](alert-system.md)).

## Section Collapse Behaviour

All three sections (critiques, gaps, overlaps) start **collapsed** by default. Each has an independent toggle:

```typescript
const [critiquesOpen, setCritiquesOpen] = useState(false);
const [gapsOpen, setGapsOpen] = useState(false);
const [overlapsOpen, setOverlapsOpen] = useState(false);
```

When all sections are collapsed, a hint appears: *"Tap sections above to expand details"*.

## Backend Source

The audit report is generated by `ContentGuardianAgent.audit_committee()` in:

- Backend: `backend/services/intelligence/agents/specialized/content_guardian.py`
- Logged by: `backend/services/today_workflow_service.py` lines 512–563
- Event type: `quality_audit`
- Alert persistence: `activity.create_alert()` with `dedupe_key` prefix `guardian:`
